Operating System-Specific Notes

Important

This section applies to version 3.6 of GNU Health.

openSUSE

Note

This section provides the instructions for the official and standard (vanilla) installation on openSUSE. If you want to use the openSUSE package based installation, please refer to the Community Pages

Download and install the Operating System

  • Download the openSUSE Leap Network CD image

  • Check the partitioning and FS options (we use ext4 filesystem)

  • Select SERVER (text only) installation

  • Enable SSHD server

  • Create the user “gnuhealth” when prompted at installation time.<br>

Install the requirements

1sudo zypper in patch gcc libxml2-devel  postgresql postgresql-server unoconv python3-pip python3-devel

Initialize the PostgreSQL environment. The next systemctl start command will generate the initial pg cluster.

1systemctl start postgresql

Update locally pip3

1su - gnuhealth
2pip3 install --upgrade --user pip

Creating the Operating System User

Debian

Important

This section applies to version 3.6 of GNU Health.

Download and install the Operating System

  • Download the Debian OS image

  • Check the partitioning and FS options (we use ext4 filesystem)

  • Deselect the “Debian desktop environment” if you just want a server (no graphical interface)

  • Enable SSHD server

  • Create the user “gnuhealth” when prompted at installation time.<br>

Install the requirements

1apt-get install postgresql patch python3-pip unoconv

Creating the Operating System User

FreeBSD

Important

This section applies to version 3.6 of GNU Health.

At Operating System installation

  • Select SSHD

  • Create the gnuhealth user at installation time

Install requirements

1pkg install postgresql13-server wget bash py37-pip \
2py37-lxml py37-pillow patch rust

Initialize PostgreSQL

1/usr/local/etc/rc.d/postgresql oneinitdb
2sysrc postgresql_enable=yes
3service postgresql start

Create Python3 links

1ln -si /usr/local/bin/python3.7 /usr/local/bin/python3
2ln -si /usr/local/bin/python3 /usr/local/bin/python

Change /bin/bash to /usr/local/bin/bash

The first line of script that starts gnuhealth (start_gnuhealth.sh) is pointing to /bin/bash. On FreeBSD you have to change that to /usr/local/bin/bash.

CentOS

Install Python 3.8

1yum install python3
2yum install python3-devel

Install PostgreSQL 12

1yum -y install https://download.postgresql.org/pub/repos/yum/reporpms/EL-7-x86_64/pgdg-redhat-repo-latest.noarch.rpm
2yum -y install epel-release yum-utils sudo yum-config-manager ^^enable pgdg12 sudo yum install postgresql12-server postgresql12
